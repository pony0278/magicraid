# 魔法基地突襲 — 設計統整

> 一句話定位:一款可愛、低暴力、節奏快的魔法冒險休閒遊戲。玩家經營自己的法師聖所、布置防禦、突襲別人的聖所;先以 Poki 輕量版驗證市場,再用同一顆核心長成 Steam／手機的深度版。

---

## 1. 總體架構:一套核心,兩種包裝

中間放一顆乾淨的核心,上面套兩件衣服。共用的是引擎與資料格式,不同的是外層的殼與調校。

**共用核心引擎(兩版逐字相同)**
- 確定性的回合制格子戰鬥
- 基地 = 一張可設計的格子佈局
- 法術／陷阱／敵人 = 資料驅動的設定檔
- 回放與結果驗證

**Poki 輕量殼**
- 一場突襲 2–3 分鐘、即點即玩、廣告營收
- 小基地、裝飾優先、精選少量法術
- 薄薄一層 PvE 引導用來解鎖法術

**深度殼(Steam／手機)**
- 完整 roguelike 冒險、大地圖策略層
- 打野獵王、流派養成、排位 PvP
- 帳號、長線進度、深度損失／護盾經濟

工程上的關鍵紀律:**核心做成資料驅動**。戰鬥是一個純函式——輸入(格子狀態 + 操作序列 + 種子),輸出新狀態。兩個殼讀同一份內容定義,深度版是「核心 + 更多內容 + 更長外層迴圈」,而不是另寫一套。

---

## 2. 戰鬥核心機制(回合制格子)

判準先行:**耐玩 = 系統互相相乘,不是功能互相堆疊。** 技能多、行動點多不會讓遊戲變深,只會變雜。深度來自「每回合都有一個有張力的取捨」以及「機制 A 會放大機制 B」。

- **共用行動點**:移動與技能搶同一池,逼出「跑位 vs 輸出」的取捨。
- **蓄力**:沒花完的點數可留到下回合(設上限),製造節奏感。
- **確定性結算**:拿掉閃避骰子,傷害是定值或最小化 RNG(要隨機就用種子)。好處同時成立:PvP 公平、回放可驗證、戰鬥更像可解的謎題。
- **位置即武器**:八方向、卡位、地形與危險格、推拉。移動機制的豐富度 = 防守設計的深度上限。
- **技能 = 互乘引擎**:狀態×地形(油 + 火)、位移類(推/拉/撞陷阱)、流派養成(讓收集到的法術長出打法)。
  - 要避開:純數值技能、嚴格優於別人的技能、無上限 RNG。
- **元素相剋三角**(火 > 自然 > 水 > 火):疊在格子戰鬥之上的易懂策略層,對年輕休閒受眾非常友善,但不取代格子戰鬥。
  > **⚠ 已更新,以 §10 為準**:此處「火 > 自然 > 水 > 火」的相剋三角已被 §10〈Demo 1 法術池〉的「**元素=機制**」模型取代——元素改為五種(火/電/土/水/中性),每種是一種會改變格子局面的機制,剋制靠機制互動湧現(水滅火、電過水放大、土牆接地擋電)而非相剋表。本段保留作為設計沿革;「**只當一層、不取代格子戰鬥**」的原則不變。

**最小可玩戰鬥組合(拿去做原型)**
- 每回合 3 點行動;移動 1 格 = 1 點,普攻 = 1 點,推 1 格 = 2 點
- 一種危險格:尖刺
- 一種狀態交互:油格 + 火
- 兩種敵人:一種貼臉近戰、一種你會想推走的

光是這幾樣彼此互乘,就已經有「直接打 vs 推進陷阱 vs 引火燒一片」的選擇張力。手感對了再加廣度。

## 3. 基地設計:動線分層

核心原則:**把「給人看的」和「擋住人的」放在動線的不同段**,炫耀就不會跟防守搶同一批格子,兩種情緒也能在輸贏兩種結局都被滿足。

進攻方動線依序經過三層:

1. **門面層**(戰利品與裝飾)→ 防守方炫耀。進攻方一定經過,不管輸贏都被看見;同時當誘餌,門面越肥代表越值得打。
2. **金庫關卡**(陷阱與召喚守軍)→ 防守方防禦。競技優化區,允許醜但有效。
3. **核心金庫**(真正的戰利品)→ 進攻方目標。

**回放是炫耀的投遞系統**:守住 → 防守方看敵人死在陷阱連段;被破 → 進攻方拿征服感、防守方仍被看見;進攻失敗 → 對方看到「敗給某某的設計」。

**損失／護盾經濟**:偷太多防守方不敢上線炫耀,偷太少進攻方沒動力。起手式為「走資源池、不直接偷核心 + 被劫後護盾 + 段位配對」。數字先做成可調參數、不要寫死,等有玩家資料再轉。

## 4. 世界結構:大地圖 + 三種節點

**大地圖是把三個活動黏成一個世界的連接組織**,概念上是核心(但開發順序仍排在戰鬥核心之後)。

地圖上只有三種節點,但全跑同一顆戰鬥引擎:

- **你的基地** — 蓋與守
- **野區地牢** — PvE,撿法術變強(2–3 層,一層 = 幾個格子房間 + 關主)
- **別人的基地** — PvP 突襲

**關鍵整合:野區副本與 PvP 突襲是同一個活動**,都是「進到一連串格子房間打過去」,差別只在佈局來源(程式生成 vs 玩家設計)。同一渲染、同一戰鬥、同一回放格式。

地圖深度兩種包裝:Poki 殼 = 輕巧的關卡選單 hub(點一下進場);深度殼 = 有移動、區域效果、領地意義的策略層。
> **待決定**:大地圖要做成「點一下進場的選單」還是「會在上面移動、有策略意義的層」?這會決定地圖投多少工、以及它在 Poki 版要不要出現。

---

## 5. PvP 取向

- **正確的簡化是「保持非同步突襲」**:打的是對方基地的存檔快照,本質是你一個人打一個玩家設計的關卡。沒有即時連線的地獄,確定性、可重播,且完整保留基地概念。
- **不要把戰鬥退化成法術對轟或剪刀石頭布**:那會讓 PvP 跟基地脫鉤——一旦兩個法師直接對幹或猜拳,玩家就沒理由蓋基地、擺陷阱,整個核心被刪掉。
- **猜拳直覺的正確用法**:做成元素相剋這「一層」,而不是整個 PvP。
  > (元素的具體形式見 §10:已從相剋三角改為機制化五元素;此處「**只做一層、不讓 PvP 塌成猜拳**」的原則完全不變,只是那「一層」現在是機制而非倍率表。)
- **法術對轟**:若想要,當成額外的小型快速模式,並刻意做小,別跟突襲搶「誰是這遊戲的 PvP」。

---

## 6. 美術與受眾(Poki 對齊)

- **可愛像素、低暴力**:敵人被「解除」成星光消散,不流血 → 過 Poki 兒童友善審查。
- **Poki 受眾特性**:年輕、休閒、平均約 15 分鐘一場、即點即玩、跑在低階裝置上。
- **Poki 審查重點**:web-first(非手機移植)、原創(非 clone)、流暢效能、一致直覺的 UX、無障礙操作、闔家友善。
- 回合制格子在技術上很輕,跑遍各裝置順暢,正好吃這套優勢。

## 7. 開發階段(Demo 0 → 5)

核心紀律:**每一階只加一樣東西、只回答一個問題,答案是 yes 才往上爬。** Demo 0 是總閘門,不過就停,後面整套都別碰。

| 階段 | 加什麼 | 驗什麼 |
|---|---|---|
| **Demo 0** 戰鬥手感 | 單法師、3 房 + 關主、推/尖刺/油火、確定性 | 分回合打起來爽不爽?(總閘門) |
| **Demo 1** 一場 run | 多個地牢、法術三選一收集、更多敵人、機制化五元素(火/電/土/水/中性,見 §10) | 「再來一局」的重玩動機在不在?法術互乘有沒有長出 build 多樣性? |
| **Demo 2** 蓋基地 + 守 | 基地編輯器(門面 + 陷阱 + 召喚守軍),先打自己的基地測 | 設計一座基地好不好玩?工具夠不夠表達?(還沒連線) |
| **Demo 3** 非同步突襲 | 佈局儲存、配對(隨機池/好友碼)、打別人基地、回放 | 突襲別人的設計爽不爽?回放炫耀有沒有打中?(第一個後端) |
| **Demo 4** 經濟與情緒迴圈 | 損失/護盾、炫耀展示、段位配對、輕量養成 | 留不留得住人?損失/保護的平衡對不對(防守方願不願意當靶)? |
| **Demo 5** 大地圖整合 | 地圖串起基地 + 地牢 + 別人基地、區域進度、(深度殼)策略層 | 三個活動有沒有黏成一體?地圖該是 hub 還是策略層(此處拍板) |

> **Demo 1 補充 — 法杖/零件**:三選一撿到的法術/零件,是**填進一把法杖**(4 槽平排、順序會影響結果),不是散裝技能。完整定案見《法杖／零件系統規格》(可作 §11)。Demo 1 只落地**附能版**(零件改相鄰法術一個機制);連結器/養杖/掠奪留待 Demo 1+/3/4。

**兩個連接點**
- **Poki 輕量殼的上線時機約在 Demo 3~4**:核心迴圈(打 → 蓋 → 突襲)已夠完整,可套輕量殼包成可上架版本先測市場,不必等整條路走完。
- **Demo 5 的地圖是兩個殼分岔處**:Poki 殼做成輕巧 hub,深度殼才長成策略層。因此「hub vs 策略層」可一路拖到 Demo 5 才靠手感拍板,前面不受影響。

**最重要的紀律**:先把 Poki 輕量殼做成 vertical slice、上線、看數據,再從同一顆核心往外長出深度殼。**不要兩個殼並行開發。** 因為核心是資料驅動的,深度版會是「加東西」而不是「重寫」。

## 8. Demo 0 詳細設計

**範圍**:單一法師,闖 3 房間 + 關主。佈局固定手工擺(不隨機)、結算確定。結構上等同一場基地突襲,所以這同時驗證了之後突襲玩法的引擎雛形。

**要證明的那一件事**:每回合都有有趣的取捨,且把連段打出來很爽。其餘一律不驗(不驗蓋基地、養成、平衡、Poki 留存)。

**三房間教學曲線(邊玩邊學,不做教學畫面)**
- **房間 1 — 上手**:純移動 + 魔法彈,兩隻小鬼,無地形。熟悉花行動點、看敵人預告、I-go-you-go。要簡單到幾乎不會死。
- **房間 2 — 推 + 尖刺**:鋪尖刺格,敵人擺在「推一下就撞上去」的位置。讓玩家自己悟出「推進陷阱 > 直接打」。← 整個 demo 成不成立的關鍵測試點。
- **房間 3 — 油 + 火 + 綜合**:鋪油窪,敵人擺在油上/油邊,火球點燃沿油蔓延;再加一格尖刺讓人串招。敵人多一點,製造壓力。

**關主 — 符文魔像**
- 約 20 HP,免疫推飛(教「有些敵人推不動」)。
- 兩回合一循環:預告(標記要砸的格子)→ 砸下 6 傷 → 砸完「過熱」一回合,受到雙倍傷害。
- 玩家循環:**看預告 → 走出標記格 → 過熱窗口把傷害全砸上去**。
- 關主房擺一塊油,過熱窗口火燒腳下油爆發 → 前面學的「油 + 火」在此有高光回報。

**數值(起點,最終靠 playtest 轉,先別寫死)**
- 法師 12 HP;每回合 3 點行動。
- 魔法彈 1 點 / 2 傷 / 遠程;推 2 點 / 推 1 格;火球 2 點 / 3 傷 / 點油蔓延。
- 小鬼 5 HP、貼臉打 3;尖刺格 6 傷;關主砸擊 6 傷。
- 「閃現」先不做(3 點全拿去移動已能走 3 格閃砸擊),playtest 嫌太勉強再加。

**死亡與通關**:法師 0 HP → 從房間 1 重來(順便測「再來一局」手感);打倒關主 = 通關。

## 10. Demo 1 法術池與流派(收集系統定案)

> 一句話定位:Demo 1 只加一個系統——**跨多種佈局的 roguelite 法術收集**;只回答一個問題——**「再來一局」的動機在不在 + 撿不同法術會不會打法不同**。基礎包驗 demo,撿取法術長流派。

### 四項定案(取代 §7 的 Demo 1 列與相關待決定)

- **三選一節奏**:每座地牢**清關後**給一次。選擇更重、分歧更慢,代價是法術池**不能有填充牌**——每一張都得是真分歧。第一座地牢做短,讓第一次分歧早點來。
- **法術欄位:上限 4**。基礎包**不算**進上限。欄位管理**只在 pick 當下發生**(拿新的就丟舊的),不做背包整理介面。打到後段「欄位滿了、得丟一個」的瞬間,就是最濃的取捨。
- **元素歸屬**:基礎法術也帶元素。**元素=機制,不是傷害色**(見下「元素系統」)——這是定案的最大調整。
- **流派呈現**:收場給一句 **run 總結**,由**實際行為計數器**生成(火燒擊殺 / 推進危險格擊殺 / 加速多打的手數 / 鏈式串殺數)。對玩家是命名,對開發者是**量測 build 多樣性的儀表板**。

---

### 元素系統:每個元素是一種「格子機制」(核心定案)

**設計主張**:不要做「火電土水」那種四種傷害顏色配一張剋制表——對休閒受眾那是負擔不是深度,也違反 §2「不做純數值」。改成**讓每個元素本身就是一種會改變格子局面的機制**;剋制關係靠**物理直覺湧現**,玩家不用背表。

五種元素(全部進 Demo 1 第一批):

- **火** — 點燃油/木、沿燃料蔓延、火格 DoT、AoE 爆發。**燃燒地形**。(已存在於核心)
- **水** — 造濕/冰格:站上去**減速**、**滅火**、**導電**(給電鋪路)。**Tempo + 地形**。
- **電** — 在相鄰敵人/導電物之間**鏈式跳躍**;**經水放大**(濕的單位/冰格導得更遠);命中可**眩暈(跳過一手)**。**擴散 + tempo**。
- **土** — 造**牆/掩體**(擋移動 + 視線、**防火**、**接地洩電**)、**定身/生根**(反機動)。**結構與封鎖**。
- **中性/物理** — 推/拉/位移,**無屬性**。

**剋制 = 機制互動湧現(不需要倍率表)**:因為元素已經是機制,連溫和倍率表都可以**不要**,剋制完全靠互動長出來,而且全是 physics 一看就懂:

- **水滅火、火蒸發水**(對立組 火 ↔ 水)
- **電過水/濕敵會放大、土牆接地擋電**(對立組 電 ↔ 土)
- **土牆防火、擋住火的蔓延與視線**
- 真要四元素關係,就用這種**對立兩組 + 互動湧現**,**絕不做四元素循環**(四個箭頭就是 RPS 負擔)。

**三道護欄(沿用 §2 精神)**:

- **位移/物理擊殺(推進尖刺/火)無視一切元素**——走位永遠是萬用解,守住「元素只是一層、不取代格子戰術」。
- **火點燃油仍是無條件原語**(教學基石),不被任何元素匹配影響。
- 元素給的是**機制優勢**(滅火、放大、接地、封鎖),不是傷害數字優勢——避免滑回 RPS。

---

### 設計紀律(沿用 §2)

- **互乘而非疊加**:不做純數值、不做嚴格更強、不做無上限 RNG。
- **元素=機制**:剋制靠互動湧現,不靠倍率表(見上)。
- **可讀性紀律(重要)**:每多一種元素會拖進新地形/狀態詞彙(水格、冰格、岩牆、麻痺…),已逼近休閒受眾的詞彙上限(參速度規格 §8)。因此**地形/狀態詞彙要極度節制**,能共用就共用(冰格沿用「減速」、岩牆沿用「牆」的心智模型)。

### 基礎包(固定,不算欄位)

- 移動(中性)/ 魔法彈(中性)/ 推(物理)/ **火球(火)**。

---

### 法術池 v1(三軸,元素為橫切標籤)

> 元素**不是第四條軸**:電/土/水 的法術照樣塞進現有三軸,讓每條軸都有「物理 + 元素」兩種工具,軸數不膨脹。

**油火/地形流 — 區域封鎖、造地形、爆發**

- **烈焰術(火)** — 前搖兩格、加號 AoE、5 傷、燒木牆、點油。賭前搖→爆發。互乘:先鋪油再轟一片;關主過熱窗口爆發;**對關主一次造成 ≥X 傷可打斷其砸擊**(沿用速度規格 §4)。*由 Demo 0 基礎包移入此處作為撿取道具。*
- **澆油(中性/燃料)** — 丟一灘油到目標格,無直接傷害。把「油+火」從撿現成變成**主動鋪設**。
- **火牆(火)** — 沿一個方向造 2~3 格火帶。封走廊、逼敵人繞路或燒穿。控場而非輸出。

**位移/控場流 — 環境擊殺、重排站位、封鎖**

- **勾索(物理)** — 把遠處敵人拉近 1~2 格(吃視線)。勾進尖刺/火,或勾過來再推。
- **震退(物理)** — 同時把**相鄰一圈**敵人各推開一格。解圍 + 一次把多隻推向危險格。
- **築牆(土)** — 升起一塊岩牆/掩體(擋移動 + 視線、防火、接地洩電)數回合。造掩體擋遠程、封走廊、改動線。**土的「造地形」招牌**。

**速度/節奏流 — tempo、先手爆發、滾雪球**

- **加速術(中性)** — 自身 +0.5 速幾手(夾 [0.5, 2.0])。自我插隊,串出額外一手。
- **冰霜(水)** — 造冰/濕格:站上去減速、**滅火**、**導電**。一張牌扛起水的整套身分(減速地形 + 滅火 + 給電鋪路)。
- **連鎖閃電(電)** — 命中後在相鄰敵人/導電物間跳躍;**經濕格/濕敵放大**;可眩暈。群體爆發,招牌連段:**冰霜鋪濕 → 閃電串一片**。

**第二批(驗證好玩後再加)**:生根(土,定身)、閃現(中性,瞬移/換位)、餘勢(中性,擊殺免費移動)、過載(電,範圍麻痺)。

---

### 元素互動 = 剋制(湧現範例,給設計與教學參考)

- **冰霜 → 閃電**:先用水鋪濕一片,閃電跳得更遠、串更多(水放大電)。
- **冰霜 → 滅掉敵方火場**:對手鋪火封路?冰一灘澆熄並反過來減速。
- **築牆擋火/視線**:岩牆擋住火的蔓延與遠程射線,也接地讓你免疫一次電擊。
- **火 vs 水地形**:火球可蒸發濕格還你乾地;反之站在水裡的你不怕被點燃。

### 新敵人 = 三軸 + 元素的考題

- **高血/裝甲**(只怕爆發)→ 烈焰/閃電串殺;或乾脆**推進陷阱**(位移無視一切)。
- **快敵**(欺負你的鏈)→ 冰霜減速 / 震退 / 餘勢。
- **遠程**(逼你找掩體)→ **築牆**切視線 / 勾索拉下來 / 閃現。
- **帶元素的敵人**:濕身敵=閃電大餐;火敵=用水澆;會放電的敵=躲岩牆後接地。每隻敵人都讓某一軸/某種元素互動變爽。

### 落地紀律

- **數字全是佔位**:前搖長度、速度幅度、火帶/冰格長度、鏈跳數都等 playtest 轉,先別寫死(資料驅動)。
- **五種元素全進第一批,但法術分批**:第一個可玩版本每軸先各 2 張、**且覆蓋全部五種元素**——建議 **烈焰術(火)、澆油(中性) / 勾索(物理)、築牆(土) / 冰霜(水)、連鎖閃電(電)** 這 6 張當首發;其餘(火牆、震退、加速、餘勢、閃現、生根、過載)當第二批。維持「一次只加必要的」紀律。
- **不要做**(沿用 Demo 1 邊界):蓋基地、聯網、長線存檔。死亡即重開,佈局重洗、選項重給。
