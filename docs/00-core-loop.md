# 0. 總覽:這遊戲是一條閉環(不是三個模式)

> 一句話定位:整個遊戲是**一條閉環**——玩家在兩種角色間來回:**解別人出的格子謎題(攻擊身分=杖)** 與 **出題給別人解(防守身分=基地)**,兩者用**同一套 set-piece 文法**。「打一場 run / 蓋基地 / 突襲」不是三個分開的模式,是這條迴圈上的三段弧。

---

## 這條環(文字版;彩現 SVG 另附)

```
①  開一場 run · 組杖 ………………………… 攻擊身分(杖)
        ↓
②  突襲:解別人的謎題 ……………………… 用杖解 set-piece
        ↓
   〔橋〕學會文法 ── 解題教你出題
        ↓
③  回家出題:布置基地 …………………… 防守身分(基地)
        ↓
④  被解:別人來突襲你的基地
        ↓
⑤  回放:看別人怎麼解你的題 …………… §3 炫耀投遞
        ↓
   〔橋〕更會解 · 更會出 ── 回放回頭教你兩端
        ↓
   ↻ 回到 ①(但這一場,你更會解、也更會出)


一套 set-piece 文法,兩種角色:
  解題側(攻擊·杖):  ① 組杖 → ② 突襲解題
  出題側(防守·基地):③ 布置 → ④ 被解 → ⑤ 回放
  兩座橋:② 解題 →〔學會文法〕→ ③ 出題;⑤ 回放 →〔更會解更會出〕→ ① 下一場
```

---

## 兩座橋是這條環的靈魂

- **學會文法(底部橋,解題 → 出題)**:玩家突襲時一直在**解** set-piece(油陣困近戰、掩體擋遠程、推進尖刺一線秒殺)。回家蓋基地,就是把這些**解過的積木反過來擺給別人**。**玩家不用學第二套詞彙**——左右兩端用的是同一套積木(危險源 / 地形 / 敵人考題 / 解法幾何),這是最深的融合。
  - **但要分清楚兩件事**:這座橋給的是**入門動機**(解過油陣,自然會反過來擺油陣),不是**精通能力**。*解*一道題只需讀懂局面;*出*一道好題要保證「可解 ∧ 公平 ∧ 難」——那是另一層技能。出題的「調難度」不是這座橋教的,是頂部橋(回放)教的;Demo 2 的「確定性 bot 試打迭代」就是在補這個落差。所以正確的承諾是「**不用學第二套詞彙,但出好題的手感靠回放長**」,而不是「永遠不用學任何新東西」。
- **更會解 · 更會出(頂部橋,回放 → 兩端皆強)**:看別人怎麼破你的基地,你同時學會**出更陰險的題**和**解更難的題**。§3 的「回放=炫耀投遞系統」在這裡讓出題有回報,而回報又回頭教你出題——這也是上面「調難度」能力的真正來源。

## 中心支點:一套 set-piece 文法

左右兩種角色用的是**同一套積木**(危險源 / 地形 / 敵人考題 / 解法幾何)。同一渲染、同一確定性戰鬥、同一回放格式;差別只在**誰出題**——演算法(野區)或玩家(基地)。這是 §4「野區副本與 PvP 突襲是同一個活動」的必然結論,也是「一核兩節點」省工的根源。

## 累積靠什麼(這條環的引擎)

run N+1 為什麼跟 run N 不一樣?因為**換一套 build,同一座佈局就變成另一道謎題**。注意這條引擎**不綁基地**:佈局來源是野區(程序生成)或基地(玩家設計)都一樣成立——所以它在 Demo 1(只有野區、還沒有基地)就能被量測,不必等到 Demo 2 才驗。累積的主從關係定死為:

- **主軸 = 每場 build(A)**:run 內三選一撿法術、組杖、滿槽取捨。這是讓「舊謎題保鮮」的唯一累積模型。
- **卷軸(B)= 收編成當下張力**:不佔槽、用掉就沒,加厚解場決策,不改累積模型。
- **養杖(C)= 收編成只養框**:跨 run 持久的只有杖的**槽位形狀**,威力每 run 重洗——給一咪咪歸屬感,又不破壞「重洗保鮮」、不製造養成雪球(守 Poki 休閒契約)。

> 一句話:**A 為主,B·C 為料。** 詳見 §10〈Demo 1 法術池〉與《法杖／零件系統規格》§11。
> 量測:換 build → 同一座佈局是否變另一道題,對應 §10「行為計數器=量 build 多樣性的儀表板」——Demo 1 就能讀數。

## 對到 Demo 階段(這條環怎麼分階長出來)

- **左半:解題(Demo 1)** — 組杖 + 突襲解題;野區用程序生成的 set-piece 出題,附能器讓組裝湧現 combo(杖有靈魂的那一步)。
- **右半的出題(Demo 2)** — 基地編輯器:玩家用同一套文法出題,確定性 bot 試打迭代「公平又難」。
- **被解 + 回放(Demo 3)** — 非同步突襲:基地存 AUDS、別人撈來打、回放投回。**情緒迴圈在這裡才真正閉環。**

所以三個 Demo 不是三個遊戲,是**這一條環分三階接起來**。

> **階段紀律提醒(別被閉環敘事拖著走)**:這條環真正閉合要到 Demo 3——出題側(③④⑤)在那之前根本不存在。也就是說 **Demo 1 全程只有「解題」半弧**,環的另一半在這一階是空的。依「每一階只加一樣、過關才往上爬」,**Demo 1 的成敗只能靠純解題 + roguelite build 多樣性撐住,不能靠「之後會閉環」這個未來價值**。看到「閉環」就想把出題提前塞進 Demo 1,是這份總覽最容易引發的誤讀——別做。
