# 魔法基地突襲 — 速度制戰鬥規格

> 用途:把回合制戰鬥的行動經濟從「行動點」改成「速度/時間制」(SPD 一系)的可動工規格。
> **本規格取代《設計統整》第 2 節中的「共用行動點 / 蓄力」設定。** Demo 0 的回合迴圈將依此重寫,原本的「每回合 3 點」不再適用。

---

## 1. 核心模型:時間鏈

不要想成「每格加能量」,想成一張**行動時刻表**:

- 每個角色有一個「下次輪到我」的**時間值**,全部從 `0` 開始。
- 規則只有一條:**時間值最小的人行動。**
- 行動完,把他的時間值 `+= 1 ÷ 速度`。
  - 快的(速度大)每次加得少 → 很快又輪到他。
  - 慢的(速度小)每次加得多 → 要等很久。
- 重複。
- **把所有人的時間值由小排到大,就是出手順序鏈**(= 玩家看到的 XCOM 式順序條)。
- 一次行動 = **做一件事**:移動一格 / 攻擊 / 施法,三選一。

---

## 2. 速度值

- **基礎速度 = 1**(所有角色)。平常就是乾淨的 1:1 交替,玩家不必理解速度。
- **加速 +0.5、緩速 −0.5**,可疊加。
- **上下限:夾在 [0.5, 2.0]**(防止緩速疊到 0 永遠不動、加速疊到無限插隊)。

| 速度 | 每次行動 time += 1÷速度 | 相對頻率 |
|---|---|---|
| 0.5(緩速) | +2.0 | 半速 |
| 1.0(基礎) | +1.0 | 基準 |
| 1.5(加速) | +0.667 | 1.5 倍 |
| 2.0(極速) | +0.5 | 2 倍 |

---

## 3. Tiebreak(確定性必備)

時間值相等時,用**固定**順序決定誰先:

1. 玩家優先。
2. 再按角色出生序(entity id 小的先)。

> 全程無 RNG、時間值為有理數累加,因此**同一種子 + 同一操作序列 → 完全相同的回放**,非同步突襲的驗證照常成立。tiebreak 一定要寫死,否則回放會對不上。

---

## 4. 動作與時間成本

**一般動作**(移動一格 / 普攻 / 輕法術):標準成本,行動後 `time += 1÷速度`。

**強力法術 = 前搖兩格制**:
- 佔順序鏈上**兩手**。
- 第一手「**蓄力中**」:鎖住,**不能移動**,鏈上顯示即將釋放。
- 第二手「**釋放**」:法術生效。
- 中間敵人會照鏈插進來行動 → 放大招 = 賭一個你撐得過前搖的安全位置(與關主砸擊的紅格預告同一個語言)。
- 兩手各自照 `1÷速度` 推進時間值。

**可打斷(已定案)**:
- 蓄力期間,施法者**受到任何傷害 → 施法被打斷**。
- 打斷後:法術不生效、蓄力那一手的節奏白費;施法者**立即解除鎖定**,下一手(原本的「釋放」)變成一般行動。**法術本身不損失、無冷卻**。
- 這讓前搖「有牙齒」:不可打斷的前搖等於沒後果,可打斷後放大招才是真正的取捨。
- **可讀性靠順序鏈**:玩家放招前看一眼鏈——蓄力到釋放之間有沒有敵人插進來、那敵人打不打得到我——就知道這發安不安全。打斷因此是看得懂的規劃決定,不是隨機懲罰。

**架構(資料驅動,為對稱性留路)**:打斷做成每個 channel 的 `可打斷` 旗標 + `打斷門檻`(預設 0 = 任何傷害都打斷;設 ∞ = 不可打斷)。

**敵人的 channel(關主砸擊)**:Demo 0 先設成不可打斷(門檻 ∞),砸擊維持純走位閃避,以隔離教學。之後(Demo 1)可給敵人/關主可打斷的 channel——例如「對關主一次造成 ≥X 傷害可打斷它的砸擊」,做出「爆發打斷大招」的爽感,且與玩家用同一套規則。

## 5. 範例推演(時間鏈 trace)

**速度相等(法 1.0、怪 1.0)** → 乾淨 1:1:
```
法(0→1) 怪(0→1) 法(1→2) 怪(1→2) …   順序:法 怪 法 怪 法 怪
```

**法師加速(法 1.5、怪 1.0)**:
```
法(0→0.67) 怪(0→1.0) 法(0.67→1.33) 怪(1.0→2.0) 法(1.33→2.0)
[2.0 平手→法優先] 法(2.0→2.67) 怪(2.0→3.0) …
順序:法 怪 法 怪 法 [法] 怪 …
```
那個連續兩個「法」就是**插隊/額外一手**。整體 1.5:1,跟速度比一致。

---

## 6. 與行動點制的差異(連段如何改變)

- **不再有「一回合內移動 + 推一氣呵成」**。現在你走一步、敵人就回應一步。
- **連段發生在交替之間**:不是你一個人擺好,而是利用敵人自己的移動 + 你的那一手。
- 走位變成核心:每一步都在跟敵人重新議價彼此的相對位置。
- **強招要付代價**:前搖期間你被鎖住、敵人會動,所以爆發是有風險的承諾,不是免費的。

## 7. Demo 0 房間 2 重設計

**舊版(行動點)**:移動 1 格 + 推(3 點)一回合把小鬼推上尖刺。← 速度制下不存在。

**新版教學目標**:在 1:1 交替下,**用走位把小鬼卡到「你—敵—危險格」一線,趁你的那一手推它撞上去。**

- 敵人 AI **會避開尖刺**(不自己踩),所以對齊不會自動發生 → 必須靠玩家走位,**這正是速度制要教的技能**。
- 教學節奏:小鬼貼上來那一手是「移動」(還沒攻擊)→ 你下一手推 → 它撞上尖刺。讓玩家學會「讓它過來、卡好角度、再一推」。

**代表性佈局(起點,精確擺位在引擎裡試)**:把尖刺放在牆角,玩家往牆角走、把小鬼引進角落,讓尖刺落在小鬼的「被推方向」上。
```
########
#s.....#   尖刺貼牆角 (1,1)
#......#
#....o.#   小鬼起點
#......#
#..@...#   法師起點
########
```
玩法:法師往左上移動,把小鬼引到 (2,2),自己站到 (3,3),推(往左上、遠離法師)→ 小鬼撞上 (1,1) 尖刺。需要 1~2 手的走位,而不是一鍵完成。

**更可控的變體(房間 3 會強化)**:火球先在你要的位置造出火格,再把小鬼推進火裡——因為火格的位置由你決定,比固定尖刺更好對齊。

## 8. 地形與障礙物

**統一心智模型(維持可讀性,玩家只要記這一句)**:可燃 = 油 + 木;火會沿可燃物蔓延;石頭安全;牆擋視線。

**兩種障礙(詞彙刻意只停在兩種)**:
- **石牆 / 石塊**:永久,擋移動 + 擋視線/攻擊,**防火**。
- **木製障礙**:擋移動 + 擋視線/攻擊,但**可燃**。

**視線(LoS)**:魔法彈/遠程攻擊需要射線,障礙物擋住射線 = 掩體。對玩家與敵人**對稱**(掩體也保護你不被遠程打)。
- 射線判定:直線經過任一阻擋格即被擋。轉角斜射建議用**嚴格規則**(角落算擋),先簡單。

**推 × 障礙**:推敵人撞牆(石或木)= 撞擊停住(小傷);也可推敵人撞敵人。

**燃燒木製障礙(復用既有油火系統)**:
- 綁定**火元素**,而非籠統的「強力」。且**需要較強的火法**才燒得穿木頭(基本火球點得著油,但燒不穿木牆)——讓木牆是真正的屏障。
- 點燃 → 「燃燒中的木牆」維持 **1 tick**(此時仍擋路、著火,等於『即將崩塌』的預告)→ 下一 tick 崩塌成**火格**(不再擋路、傷害站上去的單位),並沿相連可燃物(油、其他木牆)蔓延。
- 火格有**持續時間上限**(沿用火格計時),避免燒不完變失控。

**來源**:地牢 = 有種子的程序生成;基地 = 玩家擺放(**石牆貴而永久 vs 木牆便宜但可燃**,是防守方的材質取捨;帶火的突襲者剋木牆基地);Demo 0 = 手工。

**Demo 0 教學分配**:石牆 / 視線放前面的房間;木牆燃燒放火房間(房間 3)。一間教一件事。

> 紀律:地形詞彙就停在這兩種,別再加第三種——已逼近休閒受眾的可讀性上限。

## 9. 對其他系統的影響(cross-check)

- **確定性 / 回放 / 非同步 PvP**:全部照常(見第 3 節);地形的程序生成用種子、火蔓延無 RNG。
- **PvP 防守**:速度給防守方新工具(快守衛、緩速陷阱);地形給材質取捨(石 vs 木)。
- **可讀性**:靠第 1 節的順序鏈 + 第 8 節的統一心智模型。
- **build 多樣性**:速度成為一條法術/流派軸(時間流靠插隊串招)。
