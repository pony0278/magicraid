# 魔法基地突襲 — 設計文件總覽

> 一句話定位:一款可愛、低暴力、節奏快的魔法冒險休閒遊戲。玩家經營自己的法師聖所、布置防禦、突襲別人的聖所;先以 Poki 輕量版驗證市場,再用同一顆核心長成深度版。

這份 `docs/` 收錄專案目前所有定案文件,分「設計層」與「技術／執行層」,依「先讀什麼」排序。

> **先讀 00 看整條環,再讀 01 看細節。** [`00-core-loop.md`](./00-core-loop.md) 是總覽:整個遊戲是一條「解題↔出題」閉環(不是三個分開的模式);01 以下才是各弧的細部定案。

## 文件關係(互相參照)

```
00 核心閉環總覽 ──總覽層→ 01 設計統整(00 不改 01,只在其上加「為什麼是一條環」)
01 設計統整 ──┬─ §2 行動經濟 ──被取代→ 02 速度制戰鬥規格
              ├─ §10 元素=機制 ──被延伸→ 03 法杖／零件系統規格(§11)
              └─ §1 一核兩殼 ──落地於→ 04 技術棧定案
                                          └─ §8 B0 ──展開為→ 05 B0 遷移清單
                                                              └─ 規格來源 = demo1.html
04/02/03 ──對到產品閘門→ 06 Demo 階段計畫
全部 ──彙整追蹤於→ 07 master backlog
```

- **設計統整** 是玩法的根。其中 §2「行動點」**已被 02 速度制規格取代**;§10「元素=機制」取代了早期的相剋三角。
- **速度制戰鬥規格** 定義時間鏈、前搖打斷、地形與確定性 tiebreak,是 sim 核確定性合約(§3)的來源。
- **法杖／零件系統規格** 是 §10 的延伸(加層,不取代):三選一撿到的法術填進 4 槽平排法杖。
- **技術棧定案** 只管怎麼蓋,不重述玩法;§8 把第一步收斂成 B0。
- **B0 遷移清單** 把 `demo1.html`(已驗證好玩 = port 規格)逐項拆成 sim 核 vs view,並列出確定性風險(時間用 1/6 整數、整數 LoS/A*、種子外傳、顯式排序)。
- **Demo 階段計畫** 與 **master backlog** 把設計與技術對到可執行的進度與當前焦點。

## 核心紀律(貫穿所有文件)

- **一套核心,兩種包裝**:資料驅動的確定性戰鬥核,深度版是「加東西」而非「重寫」。
- **一個 sim,不能兩套**:sim 核唯一一份 Rust 實作,客戶端(WASM)/harness(native)/未來伺服器(cgo)全跑同一份。
- **每一階只加一樣、過關才往上爬**;Demo 0 是總閘門。
- **互乘而非疊加**:不做純數值、不做嚴格更強、不做無上限 RNG。
- **不要兩個殼並行開發**:先把 Poki 輕量殼做成 vertical slice 上線測市場。
