# 魔法基地突襲 — Demo 階段計畫

> 用途:可追進度的里程碑文件。完整設計請見《魔法基地突襲 — 設計統整》;這份只管「分幾階做、每階做完算數的標準」。

**貫穿紀律**
- 每一階只加一樣東西、只回答一個問題。
- **過關標準達成才往上爬**;沒達成就停在原地修,別硬往下做。
- 核心資料驅動,所以每一階都是「加東西」而非「重寫」——前面寫的程式後面都還用得到。
- **不要兩個殼(Poki/深度)並行開發。**

---

## Demo 0 — 戰鬥手感　🟡 設計完成,待開工

- **目標**:確認回合制格子戰鬥本身好玩。
- **加入**:單法師、3 房 + 關主、推/尖刺/油火、確定性結算。(詳細房間與關主設計見設計統整第 8 節)
- **驗證**:分回合打起來爽不爽?每回合有沒有有趣的取捨?
- **過關標準**:試玩者能在房間 2 自己悟出「推進陷阱 > 直接打」,且打完想再玩一次。
- **不要做**:蓋基地、養成、隨機生成、平衡數值、聯網。
- ⏱️ **行動經濟**:回合迴圈採速度/時間制,見《速度制規格》(取代設計統整 §2 的行動點)。
- ⚠️ **這是總閘門**。不過 → 回頭改戰鬥,後面整套都先別碰。

---

## Demo 1 — 一場 run　⚪ 規劃中

- **目標**:確認 roguelite 重玩動機 + build 多樣性。
- **加入**:多個地牢、法術三選一收集、更多敵人、**機制化五元素(火/電/土/水/中性)**。
- **驗證**:「再來一局」的動機在不在?撿不同法術會不會打法不同?
- **過關標準**:玩家自發連玩多場,且不同場打法明顯不同;能說出自己這場走了什麼流派。
- **不要做**:蓋基地、聯網、長線存檔。
- 📐 **設計依據**:法術收集/組裝見《法杖／零件系統規格》(可作設計統整 §11);元素機制與法術池見《設計統整》§10。Demo 1 只落地零件**附能版**——連結器/養杖/掠奪是 Demo 1+/3/4。

---

## Demo 2 — 蓋基地 + 守　⚪ 規劃中

- **目標**:確認「設計一座基地」好玩、工具夠表達。
- **加入**:基地編輯器(門面層 + 陷阱 + 召喚守軍);先打自己的基地或 bot 來測。
- **驗證**:設計基地好不好玩?玩家願不願意花時間調佈局?
- **過關標準**:玩家主動反覆調整佈局(想更陰險或更漂亮);不同玩家的基地長得明顯不一樣。
- **不要做**:聯網、配對、經濟。

---

## Demo 3 — 非同步突襲　⚪ 規劃中

- **目標**:確認 PvP 突襲迴圈成立。
- **加入**:佈局儲存、配對(先用隨機池或好友碼)、打別人的基地、回放。
- **驗證**:突襲別人的設計爽不爽?回放炫耀有沒有打中?
- **過關標準**:玩家會回來看自己基地被打的回放、會想再突襲/報復;打別人基地有「在解別人謎題」的感覺。
- **不要做**:複雜經濟與段位(先最陽春)。先驗「好不好玩」,不驗「平不平衡」。
- 🔌 **第一個真正需要後端的階段。**

---

## Demo 4 — 經濟與情緒迴圈　⚪ 規劃中

- **目標**:確認留存,以及損失/保護的平衡。
- **加入**:損失/護盾、炫耀展示、段位配對、輕量養成。
- **驗證**:留不留得住人?防守方願不願意當靶?
- **過關標準**:出現像樣的多日留存(玩家隔天會回來蓋/打);防守方不會因為怕被偷而把基地藏起來。
- **不要做**:把參數寫死——這一階幾乎全是調參數,資料驅動在此回本。
- 📦 **約此處可把 Poki 輕量殼包成可上架版,先去測市場。**

---

## Demo 5 — 大地圖整合　⚪ 規劃中

- **目標**:把基地、地牢、別人的基地串成一個世界。
- **加入**:地圖串起三種節點、區域進度、(深度殼)策略層。
- **驗證**:三個活動有沒有黏成一體?
- **過關標準**:玩家把它當「一個世界」在玩,而不是三個分開的模式;並用真實手感拍板「hub vs 策略層」。
- **不要做**:在 Poki 版硬塞策略層(Poki 殼 = 輕巧 hub)。
- 🔀 **兩個殼在此分岔**:Poki = hub,深度 = 策略層。所以這個決定前面都不用急。

---

## 兩個連接點

- **Poki 輕量殼上線時機 ≈ Demo 3~4**:核心迴圈(打 → 蓋 → 突襲)夠完整即可包殼上架測市場,不必等整條路走完。
- **Demo 5 是兩殼分岔處**:地圖在 Poki 是 hub、在深度版是策略層。
