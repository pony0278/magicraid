## 0. 一句話

基地 = **玩家畫的一間房**,同一顆 sim 直接拿來打(§4)。Demo 2 = **離線**的「編輯器 + 自己/bot 試打」,
只回答一個問題:**設計一座基地好不好玩、工具夠不夠表達。**

---

## 1. 範圍邊界(scope fence)

- **做**:單房基地編輯器、地形/守軍擺放、抵達核心或清場判定、**bot 試打 + 自己試打**、單一基地存檔。
- **不做**(留 Demo 3/4):聯網、配對、回放投遞、門面/裝飾炫耀層、損失/護盾經濟、多層動線、養杖。
- **🚪 過關標準**(§6):玩家**主動反覆調整佈局**(想更陰險或更漂亮);不同人的基地**明顯不一樣**。

> 紀律:**只加「出題」這一樣**。閉環的情緒回報(別人來破 + 回放)是 Demo 3,別在這階提前做。

## 2. 設計定案

### 勝利 / 失敗(突襲者視角)
- **勝利 = 抵達核心格 _或_ 清光守軍**(兩條路都算贏:硬闖 vs 打穿)。
- **失敗 = 突襲者(法師)HP ≤ 0**。
- 邊角:空基地(0 守軍)會因「清場」瞬間獲勝 → 那是爛基地,**bot 試打回報「0 手破關」**自我修正,不特判。
- 核心 = **目標格,無 HP**,踩到即贏(不是要打的東西)。

### 結構:單房(MVP)
- 一間房:**出生點(突襲者起點)→ 核心**,中間擺防守。
- 完整三層動線(門面→金庫→核心)**延後**——門面是給觀眾炫耀的,Demo 2 離線沒觀眾。

### 編輯器
- **調色盤**:石牆 / 木牆 / 油 / 尖刺 / 小鬼 / 符文眼 / **出生點(限 1)** / **核心(限 1)**。
- **操作**:點調色盤選一項 → 點格子塗;再點 = 清除(回地板)。
- **放置預算(防填滿地圖 → 無解)**:每種防守有點數成本(佔位:牆/木/油=1、尖刺=2、小鬼/眼=3;
  出生/核心/地板=0;邊界鎖牆不計),總用量 ≤ `BUDGET`(佔位 30)。超預算擋下、HUD 顯示用量。
  逼出「牆海 vs 幾隻強守軍」的取捨,對齊 §3「競技優化」。
- **可解性**:預算內仍可能把核心封死 → 靠 **bot 試打**把關(回報「卡住/無路」= 不公平,該打開)。
- **固定格子尺寸**(沿用野區房大小,~10×8)。

### 試打(Demo 2 的亮點)
- **Bot 試打**:跑 baseline agent 突襲你的基地,回報 **「N 手破關 / bot 陣亡 / 無路」**。
  即時「公平又難」迭代(§3),**複用現成 agent**,零新 AI。
- **自己試打**:當突襲者親自打一遍(複用現有戰鬥/觸控)。

### 突襲者 loadout(試打用)
- **基礎包(移動/魔法彈/推)+ 可勾選幾張威脅牌**(火球、勾索…)。
- 理由:防守要對抗的「威脅模型」可切換——例如勾選「帶火」測**木牆/油陣**會不會被火剋(§6「帶火的突襲者剋木牆基地」)。

### 存檔
- **localStorage 存一座基地**(MVP 單座,不做多槽)。**try/catch 包好**(無痕模式不崩,順手清掉 Poki 欠的無痕安全)。

### 守軍 AI
- **複用現成**(小鬼追、符文眼射)。「守在崗位附近」的 guard 行為**延後**。

### 確定性
- 基地突襲**無 RNG**(固定佈局),不需種子;同基地 + 同操作 = 同結果(回放/對拍地基照常)。

## 3. 複用 vs 新增

| 複用(已在 sim) | 新增 |
|---|---|
| 格子、地形(石/木/油/尖刺)、小鬼/符文眼 AI、戰鬥、`find_path`、突襲者(法師 + 法術)、baseline agent | **基地佈局格式**(沿用字元地圖 + 新字元 `C`=核心)、**抵達核心勝利條件**、**base-raid 模式**(贏=到核心/清場、輸=法師死)、**編輯器殼**、**localStorage 存讀** |

---

## 4. sim 需要的小改

- **核心標記**:佈局字元地圖加 `C`(核心目標格);`@`=出生點沿用。
- **base-raid 終局判定**:`terminal_status` 在 base-raid 模式下 → 法師踩到核心格 = **RaidSuccess**;`alive_enemies()==0` = RaidSuccess;法師死 = RaidFail。
  (做成 `GameState` 上的旗標或獨立進入點,實作時擇一;野區 run 的判定不變。)
- 其餘(地形/AI/戰鬥)**不動**。

---

## 5. 施工清單(MVP)

1. **sim**:`C` 核心格解析 + base-raid 終局判定(到核心/清場=贏、死=輸)。
2. **ABI**:`mr_new_base(layout)` 載入玩家基地、`mr_base_status()`;bot 試打可走 native harness 或新 ABI。
3. **harness**:`bot_raid(base) → {outcome, steps}`(複用 `choose_action` + 新終局)。
4. **殼端編輯器**:調色盤 + 點塗 + 清除 + 出生/核心限 1 + 路徑驗證。
5. **殼端模式切換**:編輯 / 試打 /(現有)野區 run。
6. **存檔**:localStorage 序列化基地(try/catch)。
7. **試打 UI**:Bot 試打鈕(回報結果)+ 自己試打入口 + 突襲者 loadout 勾選。
